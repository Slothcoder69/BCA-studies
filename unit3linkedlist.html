Linked List

Linked lists and arrays are similar since they both store collections of data.
Array is the most common data structure used to store collections of elements.

Linked lists have their own strengths and weaknesses, but they
happen to be strong where arrays are weak.

    The disadvantages of arrays are:
  • The size of the array is fixed. Most often this size is specified at compile time.
    This makes the programmers to allocate arrays, which seems "large enough"
    than required.
  • Inserting new elements at the front is potentially expensive because
    existing elements need to be shifted over to make room.
  • Deleting an element from an array is not possible.

A linked list can be define as : A linked list is a linear data structure consisting of a sequence of elements called nodes.
Each node contains two parts: data and a reference (or pointer) to the next node in the sequence.
Unlike arrays, which store elements in contiguous memory locations, linked lists use pointers to link nodes together,
allowing for efficient insertion and deletion operations at any position within the list.

    There are several types of linked lists, including:
  1. Singly linked list: Each node contains data and a reference to the next node in the sequence
  2. Doubly linked list: Each node contains data and references to both the next and previous nodes in the sequence, enabling traversal in both directions.
  3. Circular linked list: In this type of linked list, the last node points back to the first node, forming a circular structure.

"Terminology of Linked Lists"

Header Node:A header node is a special node that is attached at the beginnning
of the linked list. This header node may contain special information(metadata)
about the linked list

Data NodeThis lists contain data that store the data members and link(s) to its predecessor(and/or successor)

Head pointer "Head pointer" refers to a pointer variable that holds the memory
address of the first node in a linked list. It's essentially a reference to the
starting point of the list. The head pointer allows access to the entire linked list
by following the chain of nodes via their "next" pointers. Manipulating the
head pointer enables operations such as insertion, deletion, traversal,
and searching within the linked list.

Tail PointerSimilar to head pointer that points to the first node of a linked list,
we may have a pointer pointing to the last node of a linked list called the tail pointer
