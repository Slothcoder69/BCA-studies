Stacks and Queues

Stack: A stack is a linear data structure that follows a particular order in which operations are performed. The order may be LIFO (Last in, First out) or FILO (First in, Last out). It allows adding and removing elements in a particular order. Examples include a deck of cards, a pile of plates, etc.

It has two main operations:

  • Push: Adds an element to the collection.
  • Pop: Removes the most recently added element.

What is a stack in detail:

A stack is a linear data structure that holds a linear, ordered sequence of elements. It is an abstract data type. A stack works on the LIFO process (Last In First Out), i.e., the element that was inserted last will be removed first.

Operations on Stacks:

An operation on a stack typically refers to one of the fundamental actions you can perform on a data structure known as a stack. A stack is a collection of elements that follows the Last In, First Out (LIFO) principle, meaning that the most recently added element is the first one to be removed.

  • Push: Adds an element to the top of the stack.
  • Pop: Removes the top element from the stack.
  • Peek (or Top): Returns the top element of the stack without removing it.
  • isEmpty: Checks if the stack is empty.
  • isFull: Checks if the stack is full (relevant for implementations with a fixed size).

Stack Implementation:

A stack can be implemented using both static data structures (array) and dynamic data structures (linked list).

Array Implementation: In this implementation, a fixed-size array is used to store the elements of the stack. The top of the stack is represented by an index variable.

  • Advantages: Simple implementation, constant time access to elements.
  • Disadvantages: Limited size, resizing the array can be expensive.

Linked List Implementation: In this implementation, a linked list is used where each node contains a data element and a pointer to the next node.

  • Advantages: Dynamic size, efficient memory usage.
  • Disadvantages: Slightly more complex implementation, memory overhead for storing pointers.
