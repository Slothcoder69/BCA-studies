OOP :
Object oriented programming is a programming model in which we write programs using classes and objects utilising features of oops
Object-oriented programming is based on the concept of objects. In object-oriented programming data structures, or objects are defined, each with its own properties or attributes. Each object can also contain its own procedures or methods

Some basic concepts of OOPs are:

  • Classes
  • Objects
  • Data abstraction and encapsulation
  • Inheritance
  • Polymorphism
  • Dynamic binding
  • Message passing
    • Class:
    Collection of object is called class
    A class in C++ is a user-defined data type that has data members and member functions
    A class is like a blueprint for creating objects. An object is an instance of a class.

    Object:
    Anything that exists physically in real world
    In C++, an object is an instance of a class. A class is a blueprint that defines the data and behavior of an object.
    An object is a self-contained entity that contains both data and functions. It is a runtime entity, it is created at runtime.
    Inheritance:
    Properties of one class can be inherited to others
    There are several types of inheritance:
  • Single Inheritance: In single inheritance, a class inherits from only one base class.
  • Multiple Inheritance: In multiple inheritance, a class can inherit from more than one base class.
  • Multilevel Inheritance: In multilevel inheritance, a class inherits from another class, and then a new class is derived from it.
    • Polymorphism
    It is ability to exist in more than 1 form
    Polymorphism is one of the main features of object-oriented programming. It allows us to reuse code by creating one function that's usable for multiple uses
    We can also make operators polymorphic and use them to add not only numbers but also combine strings.
    Abstraction
    Hiding internal details and showing functionality to user
    This can be done by using access modifiers, such as private and protected, to control which members of a class can be accessed from outside the class.
    Encapsulation
    Binding of code and data together in a single unit
    Dynmaic binding
    Dynamic binding, also known as late binding, is a mechanism in C++ that allows you to call a different function depending on the type of object that is being used.
    Which code will execute is unknown until the program runs
    Message passing
    Objects communicate with each other by sending and receving information

    Benefits of OOP

  • Written code can be reused using objects and inheritance
  • to its ability to hide certain programs, It helps to build a secure system
  • Productivity :-OOP provides improved software development productivity over traditional procedural programming because of its modularity, extensibility, and reusability.
  • It is much easier to modify and maintain the program
  • Problems solving:- OOP helps programmers solve the issue without going through each and every line of the entire code. It helps in breaking the program into smaller problems and solving one error at a time.

    • DISADVANTAGES OF OOPS

  • It requires more data protection
  • Inadequate for concurrent problems
  • OOP can add complexity to your world
  • Limited reusability
  • Programs developed in oop are much larger than procedural approach
  • OOP is not a universal language , so it can't be applied everywhere

    • POP vs OOP Comparison

    POP It follows top-down approach In POP, the major emphasis is on procedure (function) and not on the data. Overloading is not possible No access specifiers It is less secure Examples of procedural oriented programming language are COBOL, FORTRAN, PASCAL, C programming language etc.
    OOP It follows bottom-up approach In OOP, the major emphasis is on data rather than procedure (function). Overloading is possible They have access specifiers It is more secure C++, Java, Dot Net, Python etc are the example of Object oriented programming (OOP) language.
