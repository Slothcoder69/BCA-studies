Polymorphism

Polymorphism: The word polymorphism means having many forms. . It as ability of a message to be displayed in more than one form

Example:
A person at the same time can have different characterstics, so the person posses different behaviour in different situation

Types of ploymorphism

  • Compile time Polymorphism
  • Run time Polymorphism

    • Compile time Polymorphism: is achived by function overloading or operator overloading


    Function overloading:    When there are multiple function with same name and different parameters then these functions are said to be overloaded.
    Functions can be overloaded by change in the number of the arguments or change in the type of the arguments


    Operator Overloading:     Operator overloading allows operators to have different meanings based on the context, especially for user-defined types. It lets you define how operators like + or * work with objects of custom classes, making code more intuitive.

    Runtime Polymorphism:
    This type of polymorphism is achieved by function overriding

    Function overriding:
    Function overridding occurs when a derived class has a definition for one of member functions of the base class. The base function is said to be overridden

    Virtual Function

    Virtual Function:
    A virtual function is a member function of the base class, that is overridden in derived class. The class that have virtual function are called polymorphic class

    Exception Handling

    In object oriented programmin(OOP), exceptions are powerful mechansim for centralised processing of errors and exceptional situatuins. THis mechanism replaces the procedure-oriented method of error handling in which each function returns a code indicating an error or scuccesful execution

    C++ Exception Classes

    In c++ standard exceptions are defined in class that we can use inside our programs.All the exception in C++ are derived from std::exception class.
    List of C++ common exception classes.

    1. std::exception:

    The base class for all standard exceptions.

    2. std::bad_alloc:

    Thrown by new when memory allocation fails.

    3. std::bad_cast:

    Thrown by dynamic_cast when it fails with a reference type.

    4. std::bad_typeid:

    Thrown by typeid when applied to a dereferenced null pointer.

    5. std::bad_function_call:

    Thrown by std::function when called on an empty function object.

    Exception handling is the indication of problem that may occur during the execution of a program
    Exception handling is the process of handling errors

    Errors can be divide into two types compile time error and run time error

    Compile time error: Error caught during compile time is called compile time errors.
    Compile time errors include library reference, syntax error or incorrect class import

    Run time error: They are also known as exceptions. An exception caught during run time create serious issues
